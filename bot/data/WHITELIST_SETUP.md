# 📋 Инструкция по использованию команды `/add_server_for_whitelist`

## 🔍 Описание

Команда `/add_server_for_whitelist` автоматически настраивает **whitelist-туннель VLESS over VLESS** для обхода блокировок BOT.

### Принцип работы

```
Клиент → Местный сервер (Россия) → Зарубежный сервер (Нидерланды) → Интернет
         ↓                        ↓
         VLESS                   VLESS + SNI Masking
```

**Трафик маскируется** под обычные HTTPS соединения с популярными доменами (Microsoft, Apple, Cloudflare и т.д.)

---

## 📝 Использование

### Формат команды

```bash
/add_server_for_whitelist <ip_local> <password_local> <name_local> <ip_foreign> <password_foreign> <name_foreign>
```

### Пример

```bash
/add_server_for_whitelist 185.200.100.1 MyPass123 Москва 45.100.200.2 EuPass456 Amsterdam
```

### Параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| `ip_local` | IP местного сервера (Россия) | `185.200.100.1` |
| `password_local` | Пароль root местного сервера | `MyPass123` |
| `name_local` | Название для местного сервера | `Москва` |
| `ip_foreign` | IP зарубежного сервера (Нидерланды) | `45.100.200.2` |
| `password_foreign` | Пароль root зарубежного сервера | `EuPass456` |
| `name_foreign` | Название для зарубежного сервера | `Amsterdam` |

---

## ⚙️ Требования к серверам

### Оба сервера должны соответствовать:

- **ОС:** Debian 11 (чистая установка)
- **ОЗУ:** минимум 2GB
- **Пропускная способность:** 1 Гбит/с
- **Доступ:** root по SSH (порт 22)
- **Состояние:** без установленных панелей управления

### Дополнительные требования:

- **Зарубежный сервер** должен иметь "чистый" IP (не в блокировках РКН)
- **Местный сервер** должен иметь стабильное соединение с зарубежным

## 🔧 Процесс настройки

Команда выполняет **5 шагов** автоматически:

### Шаг 1/5: Сканирование подсети для поиска SNI домена

- Сканируются соседние IP в подсети зарубежного сервера
- Проверяются популярные CDN домены (Microsoft, Apple, Cloudflare...)
- Измеряется задержка до каждого кандидата
- Выбирается оптимальный домен для маскировки

**Время:** 1-3 минуты

### Шаг 2/5: Установка 3X-UI на зарубежный сервер

- Установка панелей 3X-UI (версия 2.3.5)
- Настройка брандмауэра (UFW)
- Конфигурация SNI маскировки

**Время:** 2-4 минуты

### Шаг 3/5: Установка 3X-UI на локальный сервер

- Установка панелей 3X-UI
- Базовая настройка

**Время:** 2-3 минуты

### Шаг 4/5: Настройка VLESS туннеля

- Создание inbound на зарубежном сервере с REALITY и SNI
- Создание outbound на локальном сервере для туннелирования
- Генерация UUID ключей
- Настройка маршрутизации

**Время:** 1-2 минуты

### Шаг 5/5: Добавление серверов в базу данных

- Регистрация обоих серверов в боте
- Обновление конфигурации
- Перезагрузка бота (при необходимости)

**Время:** 10-30 секунд

## ⚠️ Возможные проблемы и решения

### 1. "Неверный формат IP"

**Проблема:** IP адрес указан в неверном формате

**Решение:** Используйте формат `xxx.xxx.xxx.xxx` (например, `185.200.100.1`)

### 2. "Пароль должен содержать только английские буквы и цифры"

**Проблема:** Пароль содержит специальные символы

**Решение:** Используйте только `a-zA-Z0-9`

### 3. "Один из серверов уже добавлен"

**Проблема:** Сервер с таким IP уже есть в базе

**Решение:** Используйте другой сервер или удалите существующий через `/servers`

### 4. "Ошибка установки 3X-UI"

**Проблема:** Сервер не соответствует требованиям или уже имеет панели

**Решение:** 
- Проверьте, что сервер на Debian 11
- Убедитесь, что нет других панелей (X-UI, Marzban и т.д.)
- Попробуйте переустановить ОС

### 5. "Не удалось найти оптимальный домен"

**Проблема:** В подсети нет серверов с валидными SSL сертификатами

**Решение:** Будет использован запасной домен (www.microsoft.com) - это нормально!

✅ **SNI маскировка** - трафик выглядит как посещение обычных сайтов  
✅ **REALITY протокол** - защита от активного сканирования  
✅ **TLS шифрование** - сквозное шифрование трафика  
✅ **UUID аутентификация** - уникальные ключи для каждого клиента
